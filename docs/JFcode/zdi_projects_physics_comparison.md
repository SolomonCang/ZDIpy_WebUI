# 两个 ZDI 项目在物理建模上的差异对比

## 1. 比较口径

本文对比的“两个 ZDI 项目”指的是：

1. **当前 ZDIpy_WebUI 主项目**
   - 以 `docs/brightness_map.md`、`docs/magnetic_map.md`、`docs/line_models.md`、`docs/epoch_temperature_variation.md` 为准。
   - 物理主线是：**单层光球亮度图 + 一套表面磁场球谐展开 + 盘积分谱线正演**。

2. **CTTS 方向 ZDI 项目**
   - 以 `docs/JFcode/cttszdi2_input_parameters.md` 记录的旧 `CTTSzdi2` 物理入口为历史基线。
   - 以 `docs/future/ctts_accretion_model.md` 作为未来 WebUI 版本的目标物理模型。
   - 因此这一列同时包含两层含义：
     - **旧 CTTSzdi2 已经考虑过的物理量**；
     - **未来 CTTS 双 UR 吸积模型准备显式纳入的新物理结构**。

为避免混淆，下文凡是只存在于设计方案、尚未进入当前 WebUI 主流程的内容，都会明确写成“计划引入”或“目标模型”。

## 2. 总体判断

如果只看物理目标，这两个项目的差异不是“参数多少不同”，而是**研究对象和线形成层次不同**：

- 当前 ZDIpy_WebUI 主项目面向的是**以光球为主的亮度/磁场反演**，核心观测量是 LSD Stokes I/V 轮廓，重点是大尺度表面磁拓扑和光球亮度不均匀性。
- CTTS 方向项目面向的是**磁控吸积阶段的 T Tauri 星**，需要同时描述光球、色球/吸积区、发射与吸收叠加，以及这些分量在 Stokes I/V 中不同比例地出现。
- 换句话说，当前主项目主要在做“**单层表面成像**”，而 CTTS 项目要处理的是“**多层大气与吸积结构共同成像**”。

## 3. 核心物理差异对比

| 维度 | 当前 ZDIpy_WebUI 主项目 | CTTS 方向 ZDI 项目 | 差异本质 |
|---|---|---|---|
| 研究对象 | 面向一般恒星的大尺度表面亮度与磁场反演 | 面向经典 T Tauri 星的磁控吸积系统 | CTTS 项目增加了吸积物理背景，不再只是“冷斑 + 表面磁场” |
| 大气层结构 | 默认是**单层光球主导**的局部线形成 | 历史 CTTSzdi2 已有 `Cq`/`Cm` 双图像块；未来模型进一步把它明确成**光球分量 + 色球/吸积分量** | CTTS 项目显式承认多层贡献，而当前主项目仍以单层表面图为主 |
| 线形成机制 | 主模型是 Voigt 弱场近似或单层 Unno-Rachkovsky；局部轮廓由亮度、投影、多普勒位移和磁场共同决定 | 目标模型是**双 Unno-Rachkovsky**：光球和色球都各自有一套 UR 轮廓，再在盘积分前组合 | 当前主项目关注“一个局部轮廓如何被盘积分”，CTTS 项目关注“两个物理分量如何共同生成局部轮廓” |
| 发射/吸收处理 | 主线主要针对吸收型 LSD 轮廓；亮度图通过 `b_k < 1` 表示暗区 | CTTS 模型必须允许吸积热点或色球分量形成**发射 + 吸收叠加**，`Cm` 用于表示第二分量的发射强度或覆盖度 | CTTS 项目不是单纯把暗斑做亮，而是新增一类独立发射层 |
| 反演图像变量 | 当前主线的表面标量图主要是一个亮度图 `b_k` | CTTSzdi2 历史上已区分 `Cq` 和 `Cm`；未来方案继续保留 **`Cq` 光球图 + `Cm` 色球/吸积图** | CTTS 项目在标量图像空间里天然多一个自由度，用来承载吸积层 |
| 磁场表达 | 一套球谐系数 `alpha/beta/gamma` 表示单一表面磁拓扑，可施加 Full/Poloidal/PotTor/Potential 约束 | 历史 CTTSzdi2 同样基于球谐磁场，但未来 CTTS 方案要求这套磁场同时驱动光球与色球分量，必要时再扩展到独立色球磁图 | 当前主项目默认“一套磁场对应一个线形成层”；CTTS 项目至少要考虑“一套磁场穿过两层”，甚至未来两套磁场 |
| Stokes 物理侧重 | 当前文档主线围绕 Stokes I/V；V 追踪视线磁场分量 | 旧 CTTSzdi2 在输入层允许选择 I/Q/U/V 组合，未来 CTTS 主模型仍重点落在 I/V，但更强调色球分量对 V 的直接贡献 | CTTS 项目对偏振来源的拆分更细，不再默认所有 V 只来自光球 |
| Stokes I 与 V 的权重关系 | 在单层模型中，I 与 V 主要共享同一局部大气，只是对磁场敏感程度不同 | 双 UR 方案明确要求**I 和 V 的光球/色球比例可不同**，即连续谱权重和偏振权重要分开 | 这是两个项目最关键的物理差异之一：CTTS 项目允许“看见的强度”和“看见的偏振”来自不同层次配比 |
| 时间变化 | 当前主项目已有逐历元亮度变化，但它本质上是**对单个 `Cq` 亮度图做相位调制**，并未把第二层或 dT 作为独立反演块 | CTTSzdi2 历史上已有 `V/H/L/E` 等 `Cm` 时间变化模式；CTTS 方向天然需要处理吸积区随历元变化 | 当前主项目的时间变化是“单图调制”，CTTS 项目则更接近“第二物理分量本身在演化” |
| 温度/亮度解释 | 当前实现里温度变化通过亮度映射近似处理，dT 不是独立反演变量 | CTTS 历史实现中 `Cq`、`Cm` 的时间变化和轮廓乘子会同时改变光球与色球响应，物理上更接近活动区和吸积区共同演化 | 当前主项目把温度当作亮度的代理变量；CTTS 项目更接近多区成分演化 |
| 可联合拟合的观测类型 | 主线以 LSD 谱线为核心，可附加光变曲线 | CTTSzdi2 明确区分 `Mean`、`Accr`、`phot` 等观测组，并允许单独加权 | CTTS 项目从物理上就预设了“不同观测对应不同形成层”的联合约束 |
| 谱线物理参数来源 | 当前项目把中心波长、Landé g、线强、展宽、填充因子等逐步显式配置化 | CTTSzdi2 历史上大量谱线物理量写死在源码数组里；未来 CTTS 方案则准备把这些量重新显式化 | CTTS 项目牵涉的物理参数更多，而且很多是层依赖的 |
| 差分自转 | 当前主项目使用较简化的单参数 `dOmega` 式描述 | CTTSzdi2 历史输入里使用 `beta`、`gamma` 两参数形式 | 这反映了 CTTS 旧代码在旋转动力学参数化上更接近历史 Donati/JF 体系 |
| 正则化对象 | 当前主项目主要对亮度图和磁场球谐系数做 MEM 约束 | CTTSzdi2 历史上还要处理 `Cq`、`Cm`、`B` 三块之间的相对权重；未来 CTTS 也必须保留多图像块正则 | 一旦引入 `Cm`，反演就不再是“两类量的平衡”，而是“三块物理量的平衡” |
| 适用场景 | 适合描述光球主导、发射层不显著的 LSD ZDI 问题 | 适合描述吸积、色球、热点与多层偏振共同参与的 CTTS 问题 | 两者不是同一模型的复杂度升级，而是应用场景本身不同 |

## 4. 最值得强调的五个物理分界线

### 4.1 单层表面成像 vs 双层吸积成像

当前主项目把恒星表面视为一个主要由光球控制的成像层，亮度图和磁场图都附着在这一层上。CTTS 项目则要求至少区分两个形成层：

- 光球层负责常规吸收线和基础连续谱；
- 色球/吸积层负责额外的发射、吸收、自吸收与偏振贡献。

这意味着 CTTS 项目在物理上不是“把当前亮度图改复杂一点”，而是把局部谱线形成问题从一层扩展到两层。

### 4.2 一个亮度图 vs `Cq`/`Cm` 双图像块

当前主项目的标量成像量本质上只有一个亮度图。即便加入逐历元温度变化，变化也只是附着在这个基础亮度图之上。CTTS 项目则从一开始就需要：

- `Cq`：光球亮度或吸收深度图；
- `Cm`：色球/吸积发射强度、覆盖度，或第二分量图。

这决定了 CTTS 项目能表达“亮的吸积热点不一定等于亮的光球区域”，而当前主项目不擅长表达这类分层差异。

### 4.3 Stokes V 的来源是否只来自光球

当前主项目默认的物理图像里，Stokes V 的主要来源仍是单层局部大气中的视线磁场响应。CTTS 双 UR 方案则明确要求：

- 光球可以产生自己的 V；
- 色球/吸积层也可以产生自己的 V；
- 两者在总 V 中的占比可以不同于它们在总 I 中的占比。

这一步非常关键，因为它承认了“偏振并不一定跟着总强度一起走”。

### 4.4 时间变化是调制同一层，还是让第二层真实演化

当前项目已有历元变化，但其物理含义更接近“同一个表面亮度结构在不同相位或历元被包络函数调制”。CTTS 历史模型中的时间变化则更接近：

- 吸积相关分量本身会成长、衰减或迁移；
- 第二层贡献不只是强弱变化，还会连带改变偏振与局部轮廓形态。

因此两者都叫“时间变化”，但物理内涵并不相同。

### 4.5 当前 WebUI 已实现能力 vs CTTS 目标物理

这一点必须单独说明：

- 当前 WebUI 主线已经实现的是单层亮度/磁场 ZDI 与逐历元亮度调制；
- `docs/future/ctts_accretion_model.md` 描述的是**准备引入**的 CTTS 双 UR 物理主模型；
- `docs/JFcode/cttszdi2_input_parameters.md` 则记录了旧 CTTSzdi2 曾经暴露过哪些物理入口和控制量。

也就是说，CTTS 列反映的是“**CTTS 项目希望解决的物理问题**”，而不是“当前 WebUI 已完全具备的能力”。

## 5. 一句话结论

如果用一句话概括：

**当前 ZDIpy_WebUI 主项目是在单层光球框架下重建亮度与大尺度磁场，而 CTTS 方向项目是在多层大气与磁控吸积框架下，尝试同时重建光球、色球/吸积区及其偏振贡献。**

前者的核心难点是表面磁拓扑与亮度分布，后者的核心难点则是**多层线形成、发射/吸收混合、`Cq`/`Cm` 双图像块，以及 I/V 不同来源的耦合**。

## 6. 对后续文档整理的建议

如果后续你希望把这条线继续整理成迁移路线，最值得单独展开的三部分是：

1. `Cq`、`Cm`、磁场三块量在 MEM 中如何共同正则化；
2. 双 UR 方案里 Stokes I 和 V 的权重拆分如何落到当前响应矩阵；
3. CTTSzdi2 历史参数里哪些属于“真实物理量”，哪些只是旧代码的经验开关。
